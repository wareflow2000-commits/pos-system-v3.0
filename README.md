# نظام الكاشير الذكي (Retail POS System)

## نظرة عامة
نظام كاشير ذكي متكامل مصمم لإدارة عمليات البيع بالتجزئة، المخزون، المشتريات، الموظفين، والتقارير. يعتمد النظام على معمارية "محلي أولاً" (Local-First) لضمان استمرارية العمل حتى في حالة انقطاع الإنترنت، مع إمكانيات مزامنة سحابية متقدمة.

---

## 1. ميثاق القيمة (PVC - Product Value Proposition)
يوفر النظام حلاً شاملاً لأصحاب المتاجر الصغيرة والمتوسطة من خلال:
*   **استمرارية العمل:** يعمل النظام بكفاءة بدون إنترنت بفضل قاعدة بيانات محلية (Dexie.js).
*   **المزامنة الذكية:** مزامنة تلقائية للبيانات بين الأجهزة الفرعية والسيرفر الرئيسي عند توفر الاتصال.
*   **سهولة الإدارة:** واجهة مستخدم احترافية تدعم التخصيص الكامل للهوية البصرية.
*   **التحكم الكامل:** إدارة دقيقة للمخزون، الموردين، العملاء، والرواتب في مكان واحد.

---

## 2. وثيقة متطلبات المنتج (PRD - Product Requirements Document)

### الميزات الأساسية:
*   **نقطة البيع (POS):** واجهة سريعة للبيع، دعم الباركود، الخصومات، والضرائب.
*   **إدارة المخزون:** تتبع الكميات، التنبيهات عند انخفاض المخزون، وإدارة الدفعات.
*   **إدارة المشتريات:** تسجيل فواتير الموردين وتحديث المخزون تلقائياً.
*   **إدارة الموظفين:** صلاحيات دقيقة، تسجيل الحضور، والرواتب.
*   **التقارير:** تقارير مبيعات، أرباح، ومصروفات مفصلة.
*   **الإعدادات:** تخصيص الهوية البصرية، إعدادات الطابعات، وإدارة المزامنة.

---

## 3. تجربة المستخدم وواجهة المستخدم (UI/UX)
*   **اللغة:** يدعم النظام اللغة العربية (RTL) بشكل كامل.
*   **التصميم:** تصميم عصري يعتمد على Tailwind CSS، يتميز بالوضوح وسهولة الوصول.
*   **التخصيص:** إمكانية تغيير الألوان، الخطوط، وأحجام العناصر لتناسب هوية المتجر.
*   **التفاعلية:** استخدام مكتبة `motion` لإضافة حركات انتقال سلسة وتفاعلات بصرية تعزز تجربة المستخدم.

---

## 4. معمارية النظام (System Architecture)

### التقنيات المستخدمة:
*   **Frontend:** React 18+ مع Vite، TypeScript.
*   **Local Database:** Dexie.js (IndexedDB) للتخزين المحلي.
*   **Backend:** Express.js (Node.js).
*   **Server Database:** SQLite مع Drizzle ORM.
*   **Real-time Sync:** Socket.io للمزامنة الفورية بين الأجهزة.
*   **Cloud Sync:** Supabase (اختياري).

### تدفق البيانات:
1.  **العمليات المحلية:** يتم حفظ البيانات أولاً في قاعدة البيانات المحلية (Dexie).
2.  **قائمة المزامنة (Sync Queue):** يتم تسجيل كل عملية في جدول `syncQueue` على السيرفر.
3.  **المزامنة:** يقوم `syncEngine` بمعالجة قائمة المزامنة وإرسال البيانات إلى السيرفر الرئيسي أو السحابة (Supabase) بشكل دوري.
4.  **التنبيهات:** يستخدم Socket.io لإعلام الأجهزة الأخرى بحدوث تحديثات في البيانات.

---

## 5. دليل التشغيل
*   **تثبيت المتطلبات:** `npm install`
*   **التطوير:** `npm run dev`
*   **البناء للإنتاج:** `npm run build`
*   **بدء السيرفر:** `npm start`
